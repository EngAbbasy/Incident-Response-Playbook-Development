# Incident Response Playbook

## 6 Phases

### Phase 1: Preparation
- Train employees on phishing awareness
- Configure email security tools
- Establish communication channels
- Define roles and responsibilities

### Phase 2: Identification
- Detect suspicious email
- Analyze email headers
- Confirm phishing indicators
- Create incident ticket

### Phase 3: Containment
- Block sender domain
- Isolate affected endpoint
- Disable compromised account
- Prevent lateral movement

### Phase 4: Eradication
- Remove malware
- Reset compromised credentials
- Patch vulnerabilities
- Verify threat removal

### Phase 5: Recovery
- Restore systems from backup
- Re-enable accounts
- Monitor for recurrence
- Verify normal operations

### Phase 6: Lessons Learned
- Document incident timeline
- Analyze response effectiveness
- Update playbook procedures
- Conduct team briefing

## Simulation Scenario
**Scenario**: Employee receives phishing email disguised as bank notification

**Timeline**:
1. 09:00 - Employee receives email
2. 09:15 - Employee clicks link and enters credentials
3. 09:30 - Employee reports suspicious activity
4. 09:45 - SOC Analyst confirms phishing
5. 10:00 - Playbook activated (Phase 2)
6. 10:30 - Containment initiated (Phase 3)
7. 12:00 - Eradication complete (Phase 4)
8. 14:00 - Systems recovered (Phase 5)
9. Next day - Lessons learned meeting (Phase 6)
